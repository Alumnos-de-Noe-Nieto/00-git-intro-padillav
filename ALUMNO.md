## ¿Qué es Git?
Es un sistema de control de versiones en el cual podemos ver los cambios realizados
en archivos y sirve para mantener y trabajar con versiones anteriores si algo sale mal, y nos facilita colaborar en trabajos en equipo.

## Por qué necesitamos Git
Git es necesaria para llevar el historial de cambios realizados, crear repositorios de proyectos, subir y bajar cambios de proyectos, permitir la 
colaboracion a través de la creación de ramas sin que nadie interrumpa el trabajo de otra persona, y encontrar culpables si algo falla.
## Los 3 conceptos clave de Git

1. **Repositorio:** Es la carpeta que almacena nuestro proyecto, cada archivo e imagen que tengamos, donde se guarda el historial de versiones
de un proyecto en la carpeta .git. Puede existir tanto de manera remota como local en nuestra computadora. 
2. **Commit:** Un commit es el comando que confirma y guarda de forma permanente el estado actual de los archivos en el historial del proyecto, creando un punto de retorno seguro.
3. **Rama:** Una rama funciona como una versión alterna de un proyecto, en la que una persona puede trabajar separada del proyecto original, y esta rama puede unirse al proyecto principal si es aprobada.

## Los comandos que aprendí
git init - crea un nuevo repositorio local en la carpeta actual
git clone - descarga una copia exacta de un repositorio remoto a mi computadora
git status - muestra que archivos se modificaron y cuales faltan por cambiar
git add . - prepara los cambios realizados para luego subirse
git commit -m "mensaje" - guarda los cambios con texto que describe que se hizo
git push - sube los commits locales a github
git pull - descarga y fusiona los ultimos cambios con los archivos de mi computadora
git log --oneline - muestra el historial de cambios en una sola linea por cada commit
